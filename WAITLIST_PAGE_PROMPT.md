## Technical Implementation

### Form Functionality

**When user submits email:**

**1. Validation (Client-Side):**
- Email format validation using regex or HTML5 validation
- Required field check
- Show error messages inline if invalid
- Error styling: Red border on input, red text below
- Don't allow submission if invalid

**2. Submission Process:**
- Disable button and show loading state: "Joining..." with spinner
- Submit to backend API endpoint or email service
- Don't reload page (use fetch/axios)

**3. Backend Storage Options:**

**Option A: Supabase (Recommended)**
- Create table called "waitlist"
- Store email, timestamp, optional metadata
- Enable Row Level Security
- Create API endpoint for insertion

**Option B: Email Service (Alternative)**
- EmailOctopus, Mailchimp, ConvertKit
- Use their API to add subscriber
- Can send automatic confirmation email

**Option C: Google Sheets (Quick/Simple)**
- Use Google Sheets API
- Append row with email and timestamp
- Good for MVP/testing

**4. Database Schema (if using Supabase):**

Table name: `waitlist`

Columns:
- `id` (UUID, primary key, auto-generated)
- `email` (text, unique, not null)
- `role` (text, nullable) - "artist", "producer", "venue", "fan", etc.
- `location` (text, nullable) - UK city/region
- `genres` (text array, nullable) - music genres interested in
- `referral_source` (text, nullable) - where they heard about us
- `signed_up_at` (timestamp with timezone, default: now())
- `confirmed` (boolean, default: false) - for double opt-in if needed

Indexes:
- Index on `email` for fast lookups
- Index on `signed_up_at` for chronological queries

**5. Success State:**
- Show success message: "üéâ You're on the list! Check your email for confirmation."
- Change button to green checkmark or success state
- Clear email input field
- Option: Show confetti animation (brief, subtle)
- Option: Redirect to thank-you page after 2 seconds

**6. Error Handling:**

Handle these scenarios:
- **Email already exists:** "You're already on the waitlist! We'll notify you when we launch."
- **Network error:** "Oops! Something went wrong. Please try again."
- **Invalid email format:** "Please enter a valid email address."
- **Server error:** "Unable to process request. Please try again later."

Display errors:
- Inline below input field
- Red text color
- Don't use browser alert() or confirm()
- Keep error message visible until user fixes issue

**7. Optional: Confirmation Email:**

If email service integrated, send automatic confirmation:

## Accessibility

### WCAG 2.1 AA Compliance

**Color Contrast:**
- Text on background: Minimum 4.5:1 ratio
- Large text: Minimum 3:1 ratio
- Test with contrast checker tools

**Keyboard Navigation:**
- All interactive elements keyboard accessible
- Logical tab order
- Visible focus indicators
- Skip to content link (optional)

**Screen Readers:**
- Semantic HTML (proper heading hierarchy)
- Alt text for all images
- ARIA labels where needed
- Form labels associated with inputs

**Forms:**
- Label every input field
- Error messages announced to screen readers
- Success messages announced
- Required fields indicated

**Motion:**
- Respect prefers-reduced-motion
- Provide option to disable animations
